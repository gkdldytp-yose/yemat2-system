1. ﻿예맛 통합 생산관리 시스템 소개 PPT

2. #1. 표지
   - 예맛 통합 생산관리 시스템
   - 생산, 자재, 발주, 물류를 하나로 연결한 운영 플랫폼
   - 발표자: 
   - 일자: 
   - 이 시스템은 생산 현장, 부자재 관리, 발주 관리, 물류 수불 관리를 하나의 흐름으로 연결한 내부 운영 시스템입니다.
   - 작업장별 재고와 물류 허브 재고를 분리 관리하면서, 실제 생산과 입출고 이력을 같은 데이터 위에서 처리하는 것이 핵심입니다.

3. #2. 왜 이 시스템이 필요한가
   - 생산, 자재, 발주, 입고, 불출 정보가 분산되면 운영 속도가 떨어짐
   - 작업장별 재고와 실제 물류 재고를 분리하지 않으면 오차가 커짐
   - 발주 이후 입고, 입고 이후 불출, 생산 이후 재고 차감까지 연결돼야 함
   - 관리자와 작업자에게 필요한 알림이 달라서 역할별 화면이 필요함
   - 기존 운영은 생산 계획, 자재 재고, 발주 진행, 실제 입고, 작업장 지급이 각각 따로 관리되기 쉽습니다.
   - 이 시스템은 그 단절을 줄이고, 자재 흐름을 요청부터 입고, 지급, 생산 사용까지 한 줄로 이어주는 것이 목적입니다.

4. #3. 시스템 한눈에 보기
   - 사용자/권한 관리
   - 생산 스케줄 관리
   - 생산 실적 관리
   - 원초 관리
   - 부자재 관리
   - 발주 관리
   - 물류 수불 관리
   - 알림 및 출력 기능
   - 단순 조회 시스템이 아니라 실제 현장 처리를 위한 실행형 운영 시스템

5. #4. 사용자 권한 구조
   - 최고 관리자: 전체 관리, 회원 승인, 통합 관리
   - 생산 담당: 생산 계획 및 생산 실적 처리
   - 구매 담당: 발주 등록 및 발주 현황 관리
   - 물류 담당: 입고 완료, 불출 완료/반려, 물류 허브 운영
   - 일반 작업장 사용자: 작업장 기준 자재 조회, 불출/반출 요청
   - 같은 시스템을 사용하지만 권한에 따라 보는 메뉴와 처리 가능한 액션이 다릅니다.
   - 그래서 필요한 사람에게 필요한 정보만 노출되도록 설계했습니다.

6. #5. 생산 관리 기능
   - 생산 스케줄 등록 및 조회
   - 생산 진행/완료 처리
   - 실제 생산 박스 수량 기록
   - 원재료, 부자재, 포장재 사용량 입력
   - 생산 완료 시 출력물 자동 생성
   - 생산은 계획만 관리하는 것이 아니라, 실제 사용량과 결과물까지 기록합니다.
   - 완료된 생산 건은 출력 문서로 바로 활용할 수 있도록 연결했습니다.

7. #6. 자재 관리 구조
   - 원초와 부자재를 분리 관리
   - 부자재는 로트 기반 관리 지원
   - 입고일, 소비기한, 공급업체, 단가, 최소재고 관리
   - 작업장별 재고와 물류 재고를 구분
   - 총재고는 물류 + 작업장 재고 개념으로 확장 가능
   - 단순히 품목 마스터만 있는 것이 아니라, 실제 로트와 재고 이동을 고려한 구조입니다.
   - 재고 수량만이 아니라 어떤 입고분을 썼는지까지 추적할 수 있는 기반이 있습니다.

8. #7. 발주 관리 프로세스
   - 최소재고 기준으로 발주 필요 품목 확인
   - 발주 등록 시 입고 예정일 필수
   - 발주중 / 입고완료 상태 관리
   - 입고 완료 시 물류 재고로 자동 반영
   - 입고 예정일 변경 이력 관리 가능
   - 발주 관리의 핵심은 등록 자체보다 이후 흐름입니다.
   - 이 시스템은 발주 후 실제 입고 처리와 재고 반영까지 연결하는 구조를 갖고 있습니다.

9. #8. 물류 수불 관리 프로세스
   - 물류 허브 재고 별도 관리
   - 작업장이 물류에 불출 요청
   - 물류 담당자가 실제 지급 수량 입력 후 완료 또는 반려
   - 작업장 반출 요청 시 물류로 재고 복귀 가능
   - 물류 재고와 작업장 재고 이동이 동시에 반영
   - 가장 차별화되는 부분은 물류 허브 개념입니다.
   - 입고된 자재가 바로 작업장 재고로 가지 않고, 물류에 쌓인 뒤 요청에 따라 이동되는 구조입니다.

10. #9. 알림 기능
   - 요청자에게 진행 상태 알림 제공
   - 물류 담당자에게 입고/불출 처리 대기 알림 제공
   - 작업장 사용자에게 최소재고 도달 알림 제공
   - 최고 관리자에게 회원가입 승인 대기 알림 제공
   - 사용자 권한과 작업장 기준으로 필요한 알림만 노출
   - 알림은 단순 공지형이 아니라 업무 액션을 위한 신호입니다.
   - 누구에게 어떤 알림이 필요한지를 역할 단위로 분리한 것이 운영 효율에 중요합니다.

11. #10. 출력 및 문서화
   - 생산 완료 건 A4 출력
   - 자재 사용 현황 자동 정리
   - 원재료/부자재/포장재 사용 이력 시각화
   - 현장 문서로 바로 사용할 수 있는 형태 제공
   - 현장에서는 화면만큼 출력물도 중요합니다.
   - 그래서 생산 완료 문서를 바로 인쇄해서 전달하거나 보관할 수 있도록 구성했습니다.

12. #11. 기대 효과
   - 재고 흐름 가시성 향상
   - 발주부터 입고, 지급, 생산 사용까지 데이터 연결
   - 작업장과 물류 간 책임 구분 명확화
   - 최소재고 기반 선제 대응 가능
   - 수기/분산 관리 대비 운영 오류 감소
   - 결국 이 시스템의 목적은 운영 속도와 정확도를 같이 높이는 것입니다.
   - 재고 오차, 발주 누락, 입고 확인 지연, 현장 커뮤니케이션 비용을 줄이는 데 효과가 있습니다.

13. #12. 앞으로의 확장 방향
   - 대시보드 고도화
   - 알림 딥링크 강화
   - 원초까지 물류 허브 프로세스 확장
   - 재고 이력 분석 리포트 추가
   - 모바일 사용성 개선
   - 현재도 운영 기능은 충분하지만, 앞으로는 분석과 예측 영역으로 확장할 수 있습니다.
   - 특히 재고 추세, 발주 리드타임, 사용량 패턴 분석까지 연결하면 더 강한 시스템이 됩니다.

14. #13. 마무리
   - 예맛 통합 생산관리 시스템은
   - 생산 + 자재 + 발주 + 물류를 하나로 연결한 운영 시스템
   - 현장 실행 중심 구조
   - 역할 기반 화면과 알림 중심 설계
   - 이 시스템은 단순 관리용 프로그램이 아니라, 실제 현장 운영을 끊기지 않게 이어주는 실행 도구입니다.
   - 앞으로도 운영 흐름에 맞춰 계속 개선해 나갈 수 있는 구조로 만들고 있습니다.
